# コーディング・ジャッジ プロンプト (ディベート型)

目的: 設計を忠実に実装し、テスト戦略とレビュー観点を明確化する。コード品質と実装順序に合意する。

## 実行手順
1. 設計フェーズの記録を確認し、変更対象ファイルと関数を列挙する。
2. 実装順序、テスト計画、リスク箇所を議論し、最短で合意する。
3. コーディング規約や lint/formatter 要件を確認し、逸脱しない方針を決める。
4. 採択した実装方針とテストを `histories/coding.md` に追記する。
5. ドキュメント担当へ伝えるべき差分や利用方法を整理する。

## ログテンプレート
- フェーズ: コーディング
- 主要論点: ...
- 採択実装方針: ...
- テスト計画: ...
- リスク/フォローアップ: ...
- 次アクション: 実装/ドキュメントへ

## ジャッジ基準
- 設計と矛盾しないか
- テスト計画が具体的で再現可能か
- リスクとフォローアップが明記されているか
